# 1. 제품 요구사항

## 1.1 명세서

| User | Functional | |
| --- | --- | --- |
| 플러그 | 허브 |
| 플러그를 콘센트에 꽂는다 | 시스템 초기화 | - |
| 무선 네트워크에 연결한다 | WiFi Scan  SSID 선택  PW 입력  Hub와 네트워크 세션 연결 | -  -  -  플러그와 네트워크 세션 연결  플러그들의 이름과 주소를 DB화한다 |
| 마이크를 사용할 수 있다 | 마이크 ON  마이크 OFF  마이크 음량 조절 | -  - |
| 소리가 발생한다 | PCM demodulation  스피커 ON  재생 음량 조절 | -  -  - |
| 음성으로 집안의 가전을 제어한다 | 발화음성 수집  PCM modulation  허브로 PCM 데이터 전송  허브 응답 대기 | -  -  플러그로부터 PCM 데이터 수신  기동어 인식(ETWD)  기동어 인식음 플러그로 전달  ATOM으로 명령문 전달  [ optional ]  ATOM으로부터 명령 관련 피드백 수신(기기별 시나리오에 따라)  플러그로 명령 피드백 사운드 전달(기기별 시나리오에 따라) |
| 음성으로 제어 명령의 결과가 소리로 재생된다 | 허브로부터 기동어 인식음 수신  수신한 기동어 인식음 재생  허브로부터 수신한 명령 피드백 사운드 수신  수신한 명령 피드백 사운드 재생 |  |
| 음성으로 집안 다른 공간에 인터콤 메시지(음성 메시지)를 보낸다 | 발화음성 수집  PCM modulation  허브로 PCM 데이터 전송  허브 응답 대기 | -  -  플러그로부터 PCM 데이터 수신  기동어 인식(ETWD)  기동어 인식음 플러그로 전달  option1)  타겟공간의 플러그 주소 식별  인터콤 메세지 발화문 raw 데이터 전체를 타겟 공간의 플러그에 전송  option2)  ATOM으로 명령문 전달  인식된 발화문의 길이를 기준으로 raw 오디오에서 인터콤 메시지에 해댱되는 오디오만 추출  타겟 공간의 플러그 주소 식별  타겟 공간의 플러그에 전달하고자 하는 메시지문 오디오 전송 |
| 스피커로 집안 다른 공간에서 전달되어온 인터콤 메세지를 듣는다 | 허브로부터 인터콤 메시지문 수신  메시지문 PCM demodulation  메시지문 발화 |  |
