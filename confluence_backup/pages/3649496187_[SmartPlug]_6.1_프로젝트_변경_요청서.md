# [SmartPlug]_6.1_프로젝트 변경 요청서

# 1. 개요

스마트 플러그 과제를 진행하는 과정에서 발생한 기능 요구사항을 포함한 전체 변경 요청을 기록하고, 변경 요청에 대한 세부 및 효과, 그리고 영향도를 분석 정리함으로써 과제의 완성도 및 품질향상을 목표로 함.

\

# 2. 변경 요청 배경

8/11일 IDR에서 iLab 장 지시사항에 의해 아래와 같은 신규 요구사항 발생

- 홈 허브와 연동된 음성제어와 인터콤 기능 개별 분리 시연을 타 과제에서 기개발한 멀티 카메라 연동 과제와 묶어 하나의 통합 시나리오로 진행할 것

또한 해당 건의 하위 요구사항으로써, TL로부터 다음의 신규 요구사항 발생.

- 플러그 연동 갯수 최대 2개(시나리오별  분리 운영) → 4개로 변경

\

## 4.2 분석 결과

- 영향 분석 결과 - 불수용
  - 범위 변경, 일정 지연, 품질 risk 요소의 심각도 높음. 전체 항목 영향도 높음
- **최종 결정 - 수용**
  - 조직의 비지니스적 목표에 의해, 반드시 수용해야하는 요구사항으로 지시받은 바, 변경 요청은 "수용"하는 것으로 결정하고, 본 변경 요청사항을 수용함으로써 발생하는 일정 및 개발범위, 리소스 영향은 대응 계획을 수립하고 필요시 Risk로 관리하는 방향으로 진행한다.
