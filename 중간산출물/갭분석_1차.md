# 갭 분석 1차 (LG SW PM 역량인증)

- 작성일: 2026-06-12
- 분석 범위: SmartPlug 하위 페이지 기반 1차 분석
- 주요 근거 페이지: 3.1(3649496152), 4.2(3649496164), 4.3(3649496166), 7.5(3649496204), 10.3(3649496253)

## 1) 잘된 부분 (강점)

1. 요구사항-작업 추적 구조가 존재함
- 4.2 매트릭스에서 요구사항 47개와 WBS 연결 현황, 변경(CR-01/02) 반영 내역을 명시
- 4.3 WBS에서 REQ ID와 WBS 작업 연결 근거가 확인됨

2. 변경 이후 일정/범위 영향이 수치로 제시됨
- 7.5 대시보드에서 기준선 전후 완료율, 종료일 편차(+8일), 미완료 수량(3개)까지 수치화
- 변경 전 선행(+3개), 변경 후 지연/완료율을 분리해 설명한 점은 통제 관점에서 유효

3. 검증 관점을 명시적으로 선언함
- 4.2 문서에 V&V 관점과 DoD 기반 확인 원칙이 서술됨
- 10.3 조달 요구사항 명세서에서 요구사항-검증방법-결과(충족) 구조를 사용

## 2) 부족한 부분 (취약점)

1. KPI 수치와 관리 의사결정의 인과 연결이 약함
- 수치 제시는 있으나, 임계치/판단 기준/즉시 조치가 한 번에 보이는 구조는 부족

2. 정보 배포/이해관계자 참여 증빙이 상대적으로 약함
- 이메일 전달 언급은 있으나, 배포 주기/대상/권한/승인 로그 체계가 부족

3. 종료/교훈 근거가 분산되어 있고 단일 문서로 집약되지 않음
- 종료 승인, 교훈 반영의 원문 증빙 확보가 아직 제한적

4. 품질 기준 표준화 수준이 낮음
- DoD 언급은 있으나 테스트케이스 미작성 사유와 품질판정 기준이 심사 친화형으로 정리되지 않음

## 3) 원인 분석

1. 공식 PM 부재 상태에서 PL 중심 운영으로 문서의 목적이 실행 중심에 치우침
2. 변경통제 절차 미준수 요청 유입으로 후반부 우선순위가 납기/시연 달성으로 재편됨
3. 일부 산출물 유실로 인해 사후 인증 관점의 정합 문서가 누락됨

## 4) 심사 영향도

1. 고득점 기회
- 시스템 활용 관리 역량(50%)에서 요구사항 추적, 대시보드 기반 통제는 방어 가능

2. 감점 위험
- 커뮤니케이션 역량(20%)에서 스토리 인과가 약하면 "산출물 나열"로 보일 수 있음
- 종료/교훈, 정보배포 항목은 근거 밀도 부족 시 반복 질의에 취약

## 5) 보완안 (우선순위)

1. P1: KPI -> 판단 -> 조치 -> 결과 1페이지 작성
- 핵심 KPI 3개만 선정(완료율, 편차, 미완료 수)
- 각 KPI에 대한 임계치/판단/조치/성과를 단일 표로 제시

2. P1: REQ -> WBS -> 검증 -> 결과 추적 체인 샘플 1건 시각화
- 심사위원3 코멘트 대응용 핵심 방어 슬라이드

3. P1: 종료/교훈 패키지 1페이지 작성
- 종료 기준, 달성여부, 미완료 처리방향, 재시작 시 보완 항목을 한 장에 집약

4. P2: 정보배포/이해관계자 참여 로그 보강
- 배포 대상, 채널, 주기, 승인 주체를 표로 정리

5. P2: 테스트케이스 미작성 사유와 PoC 검증 전략 명문화
- 범위 제한, DoD 기준, 5.3 결과평가 연동 근거를 명확화

## 6) 판정 요약

- 현재 상태: "기본 방어 가능, 고득점 설계는 보완 필요"
- 즉시 실행 항목: P1 3개를 발표자료 본문 우선 반영
